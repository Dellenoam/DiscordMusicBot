# Discord Music Bot

Современный музыкальный бот для Discord на **discord.py 2.5+** и **yt-dlp**.
Воспроизводит треки с YouTube и любых других площадок, поддерживаемых yt-dlp,
с приятным интерфейсом из slash-команд и кнопок управления.

[English version →](README.md)

---

## Возможности

- **Slash-команды** для всего управления плеером
- **Голосование за пропуск** с настраиваемым порогом
- **Режимы повтора**: выкл / трек / очередь
- **Перемешивание очереди**
- **Регулировка громкости** (0–200%)
- **Автоотключение** при пустой очереди или пустом голосовом канале
- **Поиск с выбором** результата, если запрос — не прямая ссылка
- **Кнопки управления** прямо в сообщении «сейчас играет»

## Команды

| Команда | Что делает |
|---|---|
| `/play <запрос>` | Воспроизвести трек или добавить в очередь (URL или поиск) |
| `/skip` | Голосование за пропуск (админы и автор трека пропускают мгновенно) |
| `/pause` | Поставить на паузу |
| `/resume` | Снять с паузы |
| `/stop` | Остановить воспроизведение и выйти из канала |
| `/queue` | Показать очередь |
| `/nowplaying` | Текущий трек с прогресс-баром |
| `/clear` | Очистить очередь |
| `/remove <позиция>` | Удалить трек из очереди |
| `/shuffle` | Перемешать очередь |
| `/loop [режим]` | Циклически менять режим повтора или задать явно |
| `/volume <0-200>` | Громкость в процентах |
| `/help` | Список всех команд |
| `/about` | О боте и авторе |

## Установка

### Требования

- **Python 3.13+**
- **uv** — установка: [docs.astral.sh/uv](https://docs.astral.sh/uv/getting-started/installation/)
- **FFmpeg** в `PATH` (поставь через `apt`, `brew`, `choco` или скачай с [ffmpeg.org](https://ffmpeg.org))
- Discord-приложение и токен бота — создаются в [Discord Developer Portal](https://discord.com/developers/applications)

### Шаги

1. Клонируй репозиторий:
   ```bash
   git clone https://github.com/Dellenoam/DiscordMusicBot.git
   cd DiscordMusicBot
   ```

2. Установи зависимости:
   ```bash
   uv sync
   ```

3. Создай файл `.env` (см. `.env.example`):
   ```env
   DISCORD_TOKEN=your_token_here
   ```

4. Запусти бота:
   ```bash
   uv run musicbot
   ```

## Docker

В репозитории есть `Dockerfile` и `compose.yaml` с предустановленным FFmpeg.

1. Скопируй и заполни файл окружения:
   ```bash
   cp .env.example .env
   # отредактируй .env и задай DISCORD_TOKEN
   ```

2. Запусти контейнер:
   ```bash
   docker compose up -d
   ```

## Настройка через `.env`

| Переменная | По умолчанию | Описание |
|---|---|---|
| `DISCORD_TOKEN` | — | **Обязательная.** Токен бота. |
| `SKIP_VOTE_RATIO` | `0.5` | Доля слушателей в канале, нужная для пропуска (0.0–1.0). |
| `ADMIN_INSTANT_SKIP` | `true` | Админы пропускают мгновенно. |
| `REQUESTER_INSTANT_SKIP` | `true` | Заказавший трек пропускает его мгновенно. |
| `INACTIVITY_TIMEOUT` | `180` | Секунды бездействия (пустая очередь) до выхода из канала. |
| `EMPTY_CHANNEL_GRACE` | `30` | Секунды ожидания после опустения голосового канала перед отключением. `0` = отключиться сразу. |
| `ACTIVITY_NAME` | *(пусто)* | Текст статуса «Playing …». Пусто — статус не выставляется. |
| `LOG_LEVEL` | `INFO` | `DEBUG`, `INFO`, `WARNING`, `ERROR`. |
| `YDL_FORCE_IPV4` | `true` | Принудить yt-dlp использовать IPv4 (обходит IPv6-троттлинг YouTube на некоторых серверах). |
